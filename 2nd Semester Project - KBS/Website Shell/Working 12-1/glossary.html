Glossary of Terms

Abbreviations

Apple Service Diagnostics/Toolkit (ASD/AST)

Tests Apple system hardware for correct operation. Booted from a USB, the ASD/AST EFI tool performs low-level tests of hardware directly and does not require an operating system to run. The ASD/AST OS uses an OS X operating system to perform tests.
Blue Screen of Death (BSOD)

A blue screen, officially known as a "Stop Error" or a bug check, is the error screen displayed by Windows upon encountering a critical error, of a non-recoverable nature, that causes the system to crash. If configured to do so, the upon the occurence of a BSOD, the computer will perform a "core dump" and save all data in memory in the form of a minidump for later retrieval. A Technician performs a Crash Dump Analysis to identify the root cause of the BSOD.
Bezel

One of the (typically) two hinges that attach an LCD display to the a laptop's chassis.
CampusNet

Works with the Computing HelpDesk, PC Services and JMU Telecom to provide network, hardware, and software technical support to customers. See the About section on the staff page for a more comprehensive list of the services CampusNet provides.
CampusNet Technician, Level I
Entry-level CampusNet employee with the following responsibilities:
  • Work up to 10 hours/week (8am - midnight 7 days/week).
  • Contact client within 24 hours of call assignment via email and voicemail.
  • Provide on-site, telephone, and in-office problem resolution.
  • Malware removal.
  • Perform data recovery on failed hard drives.
  • Verify repairs for Dell and Apple systems under warranty.
  • Install and update operating systems (Windows/Macintosh), supported JMU applications found on the downloads site (e.g. anti-virus software), and other core applications.
  • Develop and update in-house training materials.
  • Contribute to internal documentation.

    • CampusNet Technician, Level II
    CampusNet employee with all Level I responsibilities and the following additional responsibilities:
  • Work up to 20 hours/week (8am - midnight 7 days/week).
  • Handle call escalations from Level I Technicians and the rest of the team.
  • Provide advanced troubleshooting.
  • Mentor Level I Technicians to be come Level II Technicians.

    Our go-to hardware diagnostic tool for Dell systems. Can be run from the Dell Boot menu, accessed by mashing F12 as soon as the blue DELL logo appears upon boot.

    Hardware Repair Service Request Form
    A system-specific report generated upon receipt by a HelpDesk Representative or CampusNet Technician. The report remains with a system from when the customer drops it off at Frye to when they pick it up once service has been completed.

    Malwarebytes Anti-malware (MB)
    One of the core anti-malware tools used by Technicians (used in conjunction with SB and SEP), the MB utility is used to detect and remove malicious programs from systems. Can be run in PE or in Normal Mode; it is generally not advisable to install and run MB in Safe Mode.

    Network Attached Storage (NAS)
    Refers to the networked array of hard disks used to store among: user files, up-to-date OS images, tools, and applications. The NAS facilitates customer file storage during the data recovery process, easy access to commonly-used tools, and provides a common source of OS images to support the Windows and OS X auto-installers. You can always connect to the NAS from a Windows system connected to the secure network, by going to the start menu, and entering in the text box \\cnetnas. From a Mac, go to the Finder, select the Go drop-down menu from the top of the screen, and select Connect to Server; the server address is afp://cnetnas.

    Office Drive(s)
    The term office drive refers to one of two drives in the office; a Hitachi SATA internal hard drive for PC systems or a LaCie eSATA external hard drive for Apple systems. Using PE, any Windows installation can be pushed onto the former as needed. OS X is installed on the latter. When trying to determine whether an issue is hardware- or software-related, it helps to have a hard drive with a clean OS installation. If the issue persists after booting from the office drive, which has a known-good OS, it is likely hardware-related.

    PC Services/Repair (PCS)
    The group responsible for supporting the hardware diagnostic, repair, and replacement needs of JMU's IT Services department.

    Pre-Installation Environment (PE)
    A light-weight version of Windows, booted from a USB flash drive, that is used for among malware removal, software diagnostics, and OS deployment.
  • PE is useful as a malware removal tool because often times processes associated with malware interfere with the modification or deletion of files the malware depends on for persistence. PE provides Technicians with the ability to access the a system from an external, known-good OS, allowing for the identification and removal of malware-related files while processes associated with the malware are not running.
  • PE is useful as a software diagnostic tool for the same reason; if the system cannot boot or fix itself for whatever reason, PE can provide access that might otherwise not be possible when booting from the OS on the internal system drive.
  • PE is useful for OS deployment, for it can access the network drive and deploy a clean image of an up-to-date Windows OS, while also facilitating the automatic installation of several core applications.
    • Follow these steps to boot into PE:
  • Insert PE USB drive into system.
  • Power system on; as soon as the blue DELL logo appears, proceed to step 2.
  • Mash F8 to open the Boot Loader menu; this is standard across Windows on all PCs. F2 opens the Setup menu. F12 opens the Boot Loader menu as well, but this is specific to Dell systems.
  • Select PE Build; the default is already selected for you.
  • Select the version of the build; the default is already selected for you
  • Enter build-specific password
  • System will boot into PE

    • Remedy
    Application used by staff to track calls and enter hours.

    System Status Worksheet
    Contains basic information related to a particular customer and system. Used primarily by CampusNet Technicians to document work that has been done on a system, they ensure that one Technician can pick up where another left off, helping avoid a duplication of efforts.

    Spyot Search & Destroy (SB)
    One of the core anti-malware tools used by Technicians (used in conjunction with MB and SEP), the SB utility is a spyware and adware removal program that detects and removes malicious software. SB also inclused an "Immunize" feature to block the installation of spyware before it occurs e.g. by modifying the hosts file.

    TechTool Deluxe
    Our go-to hardware diagnostic tool for Apple systems. Can be run from a CD or from the external office drive.

    To-CampusNet Rack
    Rack in the HelpDesk area that contains customer systems authorized for delivery to CampusNet, systems delivered by PCS after undergoing hardware diagnostics/repair, and CampusNet loaners returned by customers.

    [Return] To-Customer Rack
    Rack in the HelpDesk area that contains customer systems to be returned and CampusNet loaners to be given to customers.

    To-PC Repair Rack
    The rack in the HelpDesk area that contains systems to be delivered to PCS for hardware diagnostics/repair.
